**Diagnosis:** All 24 failures trace to one root cause — `backend/tests/DerasaX.Tests` runs its integration tests directly against the shared local dev Postgres database (`derasax_local`, `ConnectionStrings:cs` in `appsettings.Development.json`), which is never reset between local runs. Confirmed two distinct symptoms of the same cause:
1. **AI usage quota exhaustion** (22 tests: `AiAnalysisApiTests` 3, `AiPredictionApiTests` 6, `AiQuizDraftApiTests` 3, `Phase15ComputerVisionApiTests` 10 — CV analysis also consumes the AI quota) — tenant-1's Free plan (`MaxAiGenerationsPerMonth=100`) has accumulated hundreds of `AiUsageRecord` rows from "many prior sessions" of running this suite locally (first identified by the P0-4/P0-7 background agent: 763 rows against a 100 cap). Every failure asserts an expected success/specific-error status and gets `409 Conflict` (`PlanLimitExceededException`) instead, including the two downstream-cascade failures in `Phase15ComputerVisionApiTests` that throw `KeyNotFoundException`/`ArgumentOutOfRangeException` when a later test step tries to read data an earlier (quota-blocked) step never produced.
2. **Unique-constraint collisions from leftover fixture rows** (2 tests: `AcademicDomainTests.ParentStudent_cross_tenant_rejected_same_tenant_ok_and_many_to_many` — `duplicate key value violates unique constraint "IX_parentStudentRelationships_TenantId_ParentId_StudentId"`; `UserProvisioningApiTests.SchoolAdmin_provisions_student_who_can_login_with_returned_credential` — expects `201 Created`, gets `409 Conflict` because the literal login code it tries to provision already exists from a prior run) — same "no DB reset between runs" cause, different constraint.

**Verified NOT a regression from this pass:** none of the 24 failing tests live in any file changed in this pass (P0/P1 touched `FileMetadataService.cs`, `FileStorageService.cs`, `AnalysisService.cs` (`GetHistoryForCallerAsync` only), `StudentProgressService.cs`, `OfficeHourService.cs`/`OfficeHourDto`, `LessonMaterial*`).
